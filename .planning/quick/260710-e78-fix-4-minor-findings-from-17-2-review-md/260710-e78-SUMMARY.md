## Accomplishments
- WR-02: `OutboxRelay.poll()` now catches per-row exceptions around `outboxRowPublisher.publish(row)`, logging a warning with the row id and continuing the batch instead of letting one row's failure abort publication of the remaining claimed rows.
- WR-05: `OutboxRowPublisher.publish()` adds an explicit `catch (InterruptedException ie)` before the broad `catch (Exception ex)`, calling `Thread.currentThread().interrupt()`, incrementing `attempts`, and leaving the row `PENDING` (not `FAILED`) so the next poll re-drives it.
- IN-01: `OutboxRetentionJob.sweepSentRetention()` now guards `ttlDays < 1` at the top of the method, logging a warning naming the misconfigured value and returning before computing a cutoff — preventing a negative/zero `ttl-days` from deleting every `SENT` row.
- WR-03: `OrderEntity.rejectionReason` is now `@Lob`-mapped (TEXT/CLOB), mirroring `OutboxEventEntity.payload`, with the `length` attribute dropped and the `jakarta.persistence.Lob` import added. `REJECTION_REASON_MAX_LEN` (used by `OrderConfirmationService.MAX_REASON_LEN` for app-level truncation) was retained unchanged.
