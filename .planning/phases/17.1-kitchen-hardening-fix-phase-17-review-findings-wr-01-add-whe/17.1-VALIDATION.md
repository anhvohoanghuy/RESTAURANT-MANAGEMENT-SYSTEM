## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | JUnit 5 / Spring Boot Test (Spring Security test, WebMvc test, Kafka broker-free) |
| **Config file** | `src/test/resources/application.properties` (`spring.kafka.listener.auto-startup=false`; H2 `MODE=MySQL`) |
| **Quick run command** | `./mvnw -q test -Dtest=<ClassName>` |
| **Full suite command** | `./mvnw test` (currently 156 tests green; 46 test classes) |
| **Estimated runtime** | ~90–120 seconds (full suite) |

**H2 constraint:** the suite runs on `jdbc:h2:mem;MODE=MySQL`, which does NOT support
`FOR UPDATE SKIP LOCKED`. The outbox relay is gated off in tests
(`outbox.relay.enabled=false` / `@ConditionalOnProperty`); crash-recovery is unit-tested
with mocks so the native claim query never executes under H2.

---

## Sampling Rate

- **After every task commit:** Run the focused `./mvnw -q test -Dtest=<TouchedTest>`
- **After every plan wave:** Run `./mvnw test` (full 156+)
- **Before `/gsd:verify-work`:** Full suite must be green
- **Max feedback latency:** ~120 seconds (full suite)
