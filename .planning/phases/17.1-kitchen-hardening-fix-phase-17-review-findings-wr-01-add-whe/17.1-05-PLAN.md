<objective>
Build the shared transactional-outbox foundation (I-WR-02, part 1): the outbox_events
entity, a repository with a MySQL-8 SKIP LOCKED claim query, an in-transaction
OutboxWriter that the saga services will call, and a scheduling-gated relay poller that
publishes pending rows and flips them to SENT. New files only — no existing behavior
changes here; the saga cutover happens in plans 06 (order) and 07 (inventory).

Purpose: A crash between DB commit and the afterCommit Kafka send permanently strands an
order in PENDING_CONFIRMATION. Writing the event row in the SAME transaction as the state
change, then relaying it, closes that dual-write gap. Consumers are already idempotent
(per-eventId ledgers), so at-least-once relay delivery is safe.
Output: OutboxEventEntity, OutboxEventRepository, OutboxWriter (contract for 06/07),
OutboxConfig (@EnableScheduling + String KafkaTemplate), OutboxRelay (gated), tests.
</objective>

<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| business tx commit → Kafka send | dual-write gap closed by the outbox row sharing the commit |
| relay poll across app instances | concurrent pollers must not double-publish a row |
| stored payload → broker | server-generated JSON, not user input |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17.1-12 | Denial of Service (stranded order) | dual-write commit/send gap | mitigate | outbox row committed atomically with state; relay re-derives the send from committed rows |
| T-17.1-13 | Tampering (double publish) | multi-instance relay poll | mitigate | FOR UPDATE SKIP LOCKED per-row claim; consumers idempotent (at-least-once safe) |
| T-17.1-14 | Tampering (lost ack) | relay marks SENT before broker ack | mitigate | await send future in-tx before flipping to SENT (RESEARCH Pitfall 2) |
| T-17.1-15 | Information disclosure | payload logging | accept | relay logs row id/topic/attempts, not full payload (V7) |
| T-17.1-SC | Tampering | package installs | accept | zero new packages this phase |
</threat_model>
