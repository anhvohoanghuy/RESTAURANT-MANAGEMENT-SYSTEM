<objective>
Cut the inventory saga result event over to the transactional outbox (I-WR-02, inventory
side) and replace the inline saveAndFlush+catch idempotency idiom with the existing
REQUIRES_NEW InventoryLedgerWriter (I-WR-01, inventory side).

Purpose: InventoryReservationService publishes the OrderStockResultEvent via afterCommit
— a crash between commit and send strands the saga (the order never leaves
PENDING_CONFIRMATION). Writing it to the outbox in the reservation transaction closes the
gap; the relay (plan 05) publishes it. The inline saveAndFlush+catch marks the reservation
transaction rollback-only on a concurrent duplicate; InventoryLedgerWriter (already built
in Phase 16) isolates it in a REQUIRES_NEW transaction.
Output: reservation result routed through the outbox; ledger isolated.
</objective>
