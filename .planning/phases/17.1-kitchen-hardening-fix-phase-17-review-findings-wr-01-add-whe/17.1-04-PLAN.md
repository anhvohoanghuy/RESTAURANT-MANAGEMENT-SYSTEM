<objective>
Switch the global producer value-serializer default (application.properties line 26)
from the deprecated Jackson-2 JsonSerializer to the Jackson-3 JacksonJsonSerializer
(I-WR-05), then run the FULL suite to catch any latent Payment/Table regression.

Purpose: The Jackson-2 JsonSerializer is deprecated-for-removal in Spring for Apache
Kafka 4.0 and cannot serialize Instant the way the Jackson-3 path does — a latent
landmine. Every existing publisher injects its own explicitly-typed template using
JacksonJsonSerializer, so this line is Boot's auto-config default only; the switch is
hygiene, not a live-flow behavior change. Scope is ONE line — do NOT touch the
Payment/Table own producer configs (deferred, out of scope).
</objective>
