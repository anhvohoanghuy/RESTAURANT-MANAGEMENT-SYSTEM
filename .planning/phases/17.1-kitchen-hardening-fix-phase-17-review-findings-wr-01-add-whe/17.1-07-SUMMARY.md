# Phase 17.1 Plan 07: Inventory Outbox + Ledger-Writer Cutover Summary

**InventoryReservationService now writes OrderStockResultEvent to the transactional outbox in the same DB transaction as the reservation/rejection, and delegates the idempotency-ledger insert to the shared REQUIRES_NEW InventoryLedgerWriter instead of an inline try/catch.**

## Performance

- **Duration:** ~25 min
- **Started:** 2026-07-09T03:35Z (approx, per STATE.md session start)
- **Completed:** 2026-07-09T03:57:40Z
- **Tasks:** 2/2 completed
- **Files modified:** 2

## Accomplishments
- Closed the dual-write gap where a crash between reservation-transaction commit and the
  afterCommit Kafka send could permanently strand an order in `PENDING_CONFIRMATION`: the
  `OrderStockResultEvent` (CONFIRMED or REJECTED) is now persisted as a `PENDING` outbox row in
  the same transaction as the reservation state change, and the relay (plan 05) publishes it.
- Isolated the inventory idempotency-ledger insert into its own `REQUIRES_NEW` transaction via
  the existing `InventoryLedgerWriter`, so a concurrent-duplicate constraint violation no longer
  marks the reservation transaction rollback-only.
- Removed the now-dead `InventoryStockResultPublisher` field and `TransactionSynchronization`
  afterCommit plumbing from `InventoryReservationService`.

## Files Created/Modified
- `src/main/java/com/example/feat1/DDD/inventory_context/application/InventoryReservationService.java` - Replaced inline `saveAndFlush`+catch ledger idiom with `ledgerWriter.tryInsert(...)`; replaced `publishAfterCommit(result)` with `outboxWriter.save("INVENTORY", orderId, result.eventType(), orderStockResultsTopic, orderId.toString(), result)` inside the existing `@Transactional onOrderCreated`; removed `InventoryStockResultPublisher` field and afterCommit synchronization code.
- `src/test/java/com/example/feat1/DDD/inventory_context/application/InventoryReservationServiceTest.java` - Mocks `InventoryLedgerWriter` and `OutboxWriter` instead of `InventoryProcessedEventRepository.saveAndFlush` and `InventoryStockResultPublisher`; added a concurrent-duplicate-ledger-insert test; `capturePublished(orderId)` now asserts the full outbox call (aggregateType `INVENTORY`, aggregateId, topic, msgKey, eventType) instead of the removed publisher mock.

## Decisions Made
- `orderStockResultsTopic` is a non-final `@Value`-injected field (not a constructor parameter),
  consistent with the existing `PaymentService.qrPlaceholderBaseUrl` / `KafkaInventoryStockResultPublisher.topic`
  pattern in this codebase — `@RequiredArgsConstructor` only picks up `final` fields, so Spring
  field-injects `@Value` fields separately. Tests set it via `ReflectionTestUtils.setField`.
- Kept the outer fast pre-check (`existsByEventIdAndConsumerName || existsByOrderId`) unchanged;
  only the inner insert-and-catch step was replaced with `ledgerWriter.tryInsert`.
