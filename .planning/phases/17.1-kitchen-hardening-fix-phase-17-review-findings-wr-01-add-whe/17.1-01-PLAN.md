<objective>
Attach a whenComplete send-failure callback with error logging to all four
fire-and-forget saga/kitchen Kafka publishers (K-WR-01 + I-WR-03) so a failed broker
send is observable instead of silently lost.

Purpose: Every one of these adapters currently discards the CompletableFuture returned
by KafkaTemplate.send(...); a failed send is invisible. This is the light mitigation
(K-WR-01) for the two kitchen publishers and the direct-send hardening (I-WR-03) for the
two saga publishers. These four files are independent and this runs in Wave 1 before the
transactional-outbox cutover (plans 06/07) makes the saga publishers a secondary path —
the whenComplete remains correct either way.
Output: whenComplete error logging on all four publishers; one shared adapter test.
</objective>
