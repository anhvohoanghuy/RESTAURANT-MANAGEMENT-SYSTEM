## Decisions Made

- Single shared `outbox_events` table rather than per-context tables â€” least code, and the table has no context-specific columns (RESEARCH Open Question #1, recommended option).
- Relay awaits the Kafka send synchronously inside its own transaction (`.get(timeout, TimeUnit.SECONDS)`) before flipping status, rather than using `whenComplete` in a follow-up transaction â€” avoids a window where a row could be marked `SENT` before the broker actually acknowledged it (RESEARCH Pitfall 2).
- `MAX_ATTEMPTS = 10` and `BATCH_SIZE = 100` chosen as reasonable defaults local to `OutboxRelay`; not exposed as config properties since the plan didn't require it (only `outbox.relay.delay-ms` and `outbox.relay.enabled` are externalized, per plan's interface spec).
