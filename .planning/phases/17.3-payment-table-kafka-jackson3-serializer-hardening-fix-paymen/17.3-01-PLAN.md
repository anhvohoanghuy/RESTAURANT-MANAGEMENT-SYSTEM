<objective>
Close the latent Payment/Table Kafka runtime risk left after the global Jackson-3 serializer switch.

Payment and Table have dedicated producer factories that still override the global producer serializer
with the legacy Spring Kafka `JsonSerializer`. Both event contracts contain `Instant`, so these
factories must be aligned with the project's Jackson-3 `JacksonJsonSerializer` pattern and guarded
with focused tests.
</objective>
