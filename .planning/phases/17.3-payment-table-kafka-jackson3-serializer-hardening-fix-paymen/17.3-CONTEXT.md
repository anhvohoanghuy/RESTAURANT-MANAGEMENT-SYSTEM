# Phase 17.3: Payment/Table Kafka Jackson-3 Serializer Hardening - Context

**Gathered:** 2026-07-09
**Status:** Ready for planning
**Source:** User-reported latent runtime risk + local code inspection

<domain>
## Phase Boundary

Close the remaining producer-serializer gap left after the global Jackson-3 value serializer switch. Payment and Table define dedicated producer factories that still explicitly configure the legacy Spring Kafka `JsonSerializer`, bypassing the global `spring.kafka.producer.value-serializer=org.springframework.kafka.support.serializer.JacksonJsonSerializer` setting.

This is a runtime risk because both event contracts contain `Instant`:

- `PaymentEvent.occurredAt`
- `TableOperationEvent.occurredAt`

The fix is configuration hardening only: align these dedicated producers with the project's Jackson-3 serializer pattern and add broker-free tests that would fail if the legacy serializer returns.
</domain>

<decisions>
## Implementation Decisions

### D-01 Serializer Alignment
- `PaymentKafkaProducerConfig` and `TableKafkaProducerConfig` must use `org.springframework.kafka.support.serializer.JacksonJsonSerializer`, not the legacy `JsonSerializer`.

### D-02 Explicit Producer Factories Stay
- Keep the existing dedicated producer factory and `KafkaTemplate` beans. Do not replace them with a global auto-configured template in this phase.

### D-03 Broker-Free Verification
- Add focused config tests that instantiate the producer config classes directly and assert the configured value serializer is `JacksonJsonSerializer`.

### D-04 Runtime Serde Guard
- Add real serializer/deserializer round-trip tests for `PaymentEvent` and `TableOperationEvent` using Spring Kafka's Jackson-3 `JacksonJsonSerializer`/`JacksonJsonDeserializer`.

### D-05 Scope Fence
- Do not change Payment/Table event field names, topics, publisher send behavior, after-commit behavior, outbox behavior, or consumer/DLT wiring.

### D-06 No Dependency Changes
- Do not add Jackson modules or new dependencies. The desired path is the existing Spring Boot 4 / Spring Kafka Jackson-3 serializer.

### D-07 Regression Gate
- Focused serializer/config tests and the full Maven suite must pass.
</decisions>

<specifics>
## Specific Ideas

- Use the same import as the already-fixed producers:
  `org.springframework.kafka.support.serializer.JacksonJsonSerializer`
- Assert `ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG` equals `JacksonJsonSerializer.class`.
- Add scan verification:
  `rg -n "support.serializer.JsonSerializer|VALUE_SERIALIZER_CLASS_CONFIG, JsonSerializer.class" src/main/java/com/example/feat1/DDD/payment_context src/main/java/com/example/feat1/DDD/table_context`
- Round-trip test sample events should use fixed `Instant.parse(...)` values and assert record equality after deserialization.
</specifics>
