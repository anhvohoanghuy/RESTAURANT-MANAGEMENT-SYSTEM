# Plan 07-01: Implement Service-Only Menu Order Validation And Quote Snapshot

<objective>
Add a Menu Context application/domain service that validates order-bound dish selections from `dishId + List<toppingOptionId>` and returns a price snapshot quote for valid selections, without adding HTTP endpoints or Order/Cart persistence.
</objective>

<tasks>

1. Add request/response DTOs for menu selection validation.
   - Request input: `dishId`, `List<UUID> toppingOptionIds`.
   - Response quote: dish snapshot, selected topping snapshots, base price, toppings total, total price.
   - Acceptance: DTOs do not expose persistence entities and do not require topping group IDs from caller.

2. Add stable menu domain exception support.
   - Codes:
     - `MENU_DISH_NOT_ORDERABLE`
     - `MENU_TOPPING_NOT_ORDERABLE`
     - `MENU_TOPPING_NOT_IN_DISH`
     - `MENU_TOPPING_GROUP_REQUIRED`
     - `MENU_TOPPING_GROUP_LIMIT_EXCEEDED`
   - Acceptance: tests can assert stable code values without relying on message text.

3. Implement `MenuOrderValidationService`.
   - Load dish and category status.
   - Load all topping groups for the dish.
   - Load selected topping options by IDs.
   - Verify selected options exist, are active, and belong to the selected dish's groups.
   - Group selected options by topping group.
   - Enforce every group min/max.
   - Acceptance: no controller or public endpoint is introduced.

4. Implement quote calculation.
   - Base price comes from the dish.
   - Additional price is the sum of selected topping options.
   - Total price is base plus additional.
   - Snapshot includes current dish/topping names and prices.
   - Acceptance: valid quote can be stored by a future Order Context without re-reading menu data for historical display.

5. Add focused tests.
   - Valid selection returns expected snapshot and total price.
   - Inactive/missing dish or inactive category returns `MENU_DISH_NOT_ORDERABLE`.
   - Missing/inactive selected option returns `MENU_TOPPING_NOT_ORDERABLE`.
   - Option from a different dish returns `MENU_TOPPING_NOT_IN_DISH`.
   - Missing required group returns `MENU_TOPPING_GROUP_REQUIRED`.
   - Too many options in a group returns `MENU_TOPPING_GROUP_LIMIT_EXCEEDED`.

6. Run full Maven tests.
   - Command: `mvnw.cmd test` with local `JAVA_HOME` if needed.

</tasks>

<verification>

- Focused unit tests cover the valid quote and all stable error categories.
- Existing Menu Context and Auth tests still pass.
- Full `mvnw.cmd test` passes.

</verification>

<success_criteria>

- No HTTP endpoint is added in Phase 07.
- No Order/Cart aggregate or persistence is introduced.
- Service input stays `dishId + List<toppingOptionId>`.
- Valid selections return a price snapshot quote.
- Invalid selections throw stable menu domain exceptions.
- Strict topping group validation is enforced.

</success_criteria>
