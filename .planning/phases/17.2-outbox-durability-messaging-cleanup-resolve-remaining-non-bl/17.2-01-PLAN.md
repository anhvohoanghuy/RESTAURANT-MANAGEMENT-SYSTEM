<objective>
Close WR-01: prove (and, if broken, fix) that the transactional-outbox wire format round-trips
losslessly. `OutboxWriter` serializes events with the Spring-injected `tools.jackson.databind.ObjectMapper`;
`OutboxRelay` republishes that JSON string verbatim via `StringSerializer`; consumers deserialize
with their own `JacksonJsonDeserializer`. Because the relay is disabled in tests, NO existing test
exercises the real serialize -> publish -> deserialize path, so nothing guarantees the two mappers
agree on `Instant` and `BigDecimal` encoding.

Purpose: A silent `Instant`/`BigDecimal` encoding mismatch would poison every saga event at runtime
while every test stays green. This plan makes that path test-covered.
Output: A round-trip integration test (RED-first if a mismatch exists); mapper alignment in
`OutboxWriter`/`OutboxConfig` only if the test proves a mismatch.
</objective>
