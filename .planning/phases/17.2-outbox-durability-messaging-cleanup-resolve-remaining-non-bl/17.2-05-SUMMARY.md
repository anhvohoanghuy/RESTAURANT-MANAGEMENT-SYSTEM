# Phase 17.2 Plan 05: Rejection-Reason Constant + Ordinal-Free Kitchen Rank Map Summary

**Single `REJECTION_REASON_MAX_LEN` constant on `OrderEntity` now drives both the truncation cap and the `@Column` length (IN-03); `KitchenStatusProjectionService.deriveTargetStatus` is ordinal-free via an explicit `ITEM_RANK` map, with a reflection-based test pinning the intended ranking (WR-05).**

## Accomplishments
- Unified the previously-unaligned magic numbers (`MAX_REASON_LEN=60000` vs `@Column(length=65535)`)
  into one authoritative constant, `OrderEntity.REJECTION_REASON_MAX_LEN`. The column length is now
  unified DOWN to 60000 (matching the pre-existing effective truncation cap), so no behavior change.
- Replaced `KitchenItemStatus.ordinal()` comparisons in `deriveTargetStatus` with an explicit
  `ITEM_RANK` map (mirroring the existing `FULFILLMENT_RANK` idiom), removing the last silent-reorder
  trap in the kitchen-status projection path.
- Added a reflection-based test that pins `ITEM_RANK`'s intended ordering directly, independent of
  `KitchenItemStatus`'s enum declaration order — a genuine RED (`NoSuchFieldException`) before the map
  existed, then GREEN once introduced.
- CR-01 non-regression guard verified and preserved: `recordProcessed(event.eventId())` remains the
  literal last statement of `onTicketStatusChanged`; no `REQUIRES_NEW`/`Propagation` code was touched
  or introduced (see "Issues Encountered" for a note on the plan's grep-based acceptance check).

## Decisions Made
- Placed the single IN-03 constant on `OrderEntity` (not a separate shared holder) since it's the
  class that needs a compile-time constant for `@Column(length=...)`; `OrderConfirmationService`
  references it directly (existing dependency direction — the service already imports `OrderEntity`).
- For WR-05's TDD RED phase, chose a reflection-based test over a purely behavioral one, since a
  behavioral test would pass unchanged under both the old ordinal-based code and the new rank-map
  code (behavior is explicitly required to stay identical) and thus would not produce a real RED.
