# Plan 09-01: Implement Authenticated Order Context Cart MVP

<objective>
Add an authenticated user cart inside Order Context that validates menu/table selections through ports, stores immutable snapshots, and exposes item management APIs.
</objective>

<tasks>

1. Add Order Context domain, DTOs, and stable exception codes.
   - Cart, cart line, cart status, ports, snapshots, and API DTOs.
   - Stable errors for invalid quantity, missing line, empty/mismatched table, and cart access failures.

2. Add JPA persistence and repositories.
   - `order_carts` with one active cart per user.
   - `order_cart_lines` with stored dish/topping snapshots and totals.
   - Persist topping option IDs in normalized sorted key form for merge behavior.

3. Add Menu and Table adapters.
   - `MenuQuotePort` wraps `MenuOrderValidationService`.
   - `TableValidationPort` wraps `TableCatalogService.validateOrderableTable`.
   - Order Context does not directly own Menu/Table rules.

4. Implement cart application service.
   - `GET /cart` creates/returns active cart for current user.
   - Add item validates table and menu, stores snapshots, merges by dish + sorted topping IDs, and updates quantity.
   - Patch quantity requires positive integer.
   - Delete line removes one owned line.
   - Clear cart removes all lines for current user's active cart.
   - Cart totals are calculated from stored line snapshots.

5. Add cart controller and security.
   - Authenticated endpoints under `/cart`.
   - Use authenticated principal user id.
   - Keep no admin cart access in this phase.

6. Add focused tests.
   - Unit tests for merge behavior, quantity validation, stored totals, owner scoping.
   - Controller tests for endpoint contracts.
   - Integration tests proving token-authenticated cart operations and `/cart` owner behavior.

7. Run full Maven tests.
   - Command: `mvnw.cmd test` with local `JAVA_HOME` if needed.

</tasks>

<verification>

- Focused Order Context tests pass.
- Existing Menu, Table, and Auth tests still pass.
- Full `mvnw.cmd test` passes.
</verification>

<success_criteria>

- Order Context owns cart code and persistence.
- Menu/Table validation is consumed through Order Context ports.
- One active cart per user.
- Owner-only authenticated APIs.
- Merge by dish id plus sorted topping option ids.
- Stored snapshots drive cart reads and totals.
- No checkout/payment/order submission introduced.
</success_criteria>
