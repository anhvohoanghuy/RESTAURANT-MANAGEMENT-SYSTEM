# Plan 08-01: Implement Table Context Catalog, Public Listing, Validator, And Dev Seed

<objective>
Add a Table Context backend vertical slice for dining area/table catalog management, active public listing, and service-only table validation snapshots for future Order Context use.
</objective>

<tasks>

1. Add Table Context domain model and DTO contracts.
   - Create `DiningArea`, `DiningTable`, table catalog status, table domain exception, admin DTOs, public DTOs, and validation snapshot DTOs.
   - Acceptance: domain concepts do not include occupancy, reservations, table sessions, or Order/Cart state.

2. Add persistence entities, repositories, and adapters.
   - Store dining areas and dining tables in relational tables compatible with Spring Data JPA/MySQL.
   - Enforce unique table `code`.
   - Represent area relation from table to area.
   - Acceptance: archive/status filtering can be implemented without physical delete.

3. Implement table catalog application service.
   - Admin create/update/list/fetch/archive for areas.
   - Admin create/update/list/fetch/archive for tables.
   - Validate required fields, unique code, area existence/orderability for active table usage, and optional positive capacity.
   - Acceptance: stable domain errors are assertable by tests.

4. Implement public active table tree.
   - `GET /tables/public` returns active areas with active tables ordered by sort order/code.
   - Exclude inactive or archived areas/tables.
   - Acceptance: public response is area -> tables and does not expose admin-only/internal data.

5. Implement service-only table validator.
   - `validateOrderableTable(tableId)` returns `tableId`, `code`, `name`, `areaId`, and `areaName`.
   - Missing/inactive/archived table throws `TABLE_NOT_ORDERABLE`.
   - Missing/inactive/archived area throws `TABLE_AREA_NOT_ORDERABLE`.
   - Acceptance: no public validation endpoint is introduced.

6. Add controllers and security wiring.
   - Admin endpoints:
     - `/admin/tables/areas`
     - `/admin/tables`
   - Public endpoint:
     - `GET /tables/public`
   - Ensure admin endpoints follow existing `/admin/**` protection and public endpoint is reachable according to project security conventions.

7. Add minimal dev/test seed data.
   - Provide a few sample dining areas and tables.
   - Guard seed behavior with dev/test profile or an environment-safe property.
   - Acceptance: production startup does not require seed fixture data.

8. Add focused tests.
   - Domain/application tests for create/update/archive validation.
   - Public listing excludes inactive/archived area/table data.
   - Validator returns snapshot for active table and stable errors for invalid table/area.
   - Controller/integration tests cover admin CRUD and public listing contracts.
   - Seed behavior is covered without depending on production fixture data.

9. Run full Maven tests.
   - Command: `mvnw.cmd test` with local `JAVA_HOME` if needed.

</tasks>
