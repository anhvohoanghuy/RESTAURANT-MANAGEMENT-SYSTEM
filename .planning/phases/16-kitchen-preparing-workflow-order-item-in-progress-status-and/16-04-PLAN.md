<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OrderLinePreparingEvent payload → settlement handler | Untrusted event drives real stock deduction |
| Concurrent settlements (same order, different lines) → reservation + balance rows | Must not double-deduct, go negative, or miss/duplicate the SETTLED transition |
| Ledger insert (duplicate key) → outer settlement transaction | A collision must not poison the business transaction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-02 | Tampering/Repudiation | Replay double-deduction | mitigate | Ledger fast pre-check + REQUIRES_NEW insert (WR-01) + per-(orderId,orderLineId) settlement-guard second layer (D-03) |
| T-16-04 | Tampering | Negative stock | mitigate | Deduct on PESSIMISTIC_WRITE-locked balances; clamp on_hand and reserved to >=0 with WARN anomaly log (D-03, Pattern 4) |
| T-16-08 | Tampering | Last-line race / phantom reservation | mitigate | `lockByOrderId` (PESSIMISTIC_WRITE) makes count-vs-totalLines atomic; residual reserved force-zeroed on last line (Pitfall 2/3) |
| T-16-11 | Denial of Service | Poison-pill anomalous record (missing reservation) | mitigate | Throw on missing HELD reservation so the DefaultErrorHandler routes to the DLT instead of silent loss (D-03) |
| T-16-12 | Denial of Service | Malformed line (no recipe / unconvertible unit) | mitigate | Zero-contribution + log tolerance; never throws on recipe resolution (D-05) |
| T-16-13 | Denial of Service | Lock-order deadlock between concurrent settlers | mitigate | Ascending-ingredientId lock acquisition, same discipline as the reservation service |
| T-16-SC | Tampering | npm/pip/cargo installs | accept | No packages installed — no new dependencies |
</threat_model>
