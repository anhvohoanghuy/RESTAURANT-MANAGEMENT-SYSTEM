## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kafka broker → consumer deserialization | Untrusted bytes / headers become a typed event |
| Anomalous / poison records → DLT | Failures must isolate to a dead-letter topic, not block the partition |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-03 | Denial of Service | Poison-pill payload (malformed JSON/wrong type) | mitigate | ErrorHandlingDeserializer + DefaultErrorHandler + DeadLetterPublishingRecoverer; DeserializationException non-retryable → straight to orders.line-preparing.DLT |
| T-16-06 | Tampering/Elevation | Forged `__TypeId__` header driving arbitrary class deserialization | mitigate | USE_TYPE_INFO_HEADERS=false + forced VALUE_DEFAULT_TYPE + TRUSTED_PACKAGES allow-list (com.example.feat1.*) |
| T-16-11 | Denial of Service | Missing-reservation throw stranding a partition | mitigate | Thrown domain exception is retried (FixedBackOff) then routed to DLT rather than infinitely reprocessed |
| T-16-02b | Tampering | Concurrent-duplicate ledger poisoning the tx | mitigate | Integration test proves REQUIRES_NEW isolation (WR-01) — outer tx commits clean |
