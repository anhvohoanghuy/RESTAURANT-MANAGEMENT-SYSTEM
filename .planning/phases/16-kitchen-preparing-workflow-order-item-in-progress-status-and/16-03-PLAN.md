<objective>
Build the Order Context trigger for the kitchen preparing workflow: a staff-gated endpoint (D-02) that flips an order line to PREPARING and the order CONFIRMED→PREPARING on the first line (D-01/D-06), with state guards, and an explicit Jackson-3 producer that publishes `OrderLinePreparingEvent` (carrying `totalLines` + the prepared line's recipe payload) only after commit.

Purpose: The producer half of the saga. Placed under `/admin/orders/**` so zero `SecurityConfig` change is needed (placing it under `/orders/**` would be an authorization bug letting plain USER role trigger kitchen actions). Depends on 16-01's event contract, enums, and exception codes.
Output: `OrderLinePreparingPublisher` port, `KafkaOrderLinePreparingPublisher` adapter, `OrderLinePreparingKafkaProducerConfig`, `OrderLinePreparingService` (+ unit test), `OrderLinePreparingController`.
</objective>
