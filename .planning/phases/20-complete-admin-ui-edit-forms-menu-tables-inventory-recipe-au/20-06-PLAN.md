## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SessionsView → sessionsApi → /auth/sessions* (any authenticated) | Server enforces authentication; view is open to all roles. |
| Router adminOnly guard → authState.roles | Client-side redirect is UX only; the recipe API remains ADMIN-enforced server-side. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2006-01 | Elevation of Privilege | recipe route `adminOnly` guard | accept (UX only) | The `beforeEach` redirect stops STAFF from *seeing* the builder, but is NOT the security boundary — SecurityConfig's `/admin/menu/recipes`→ADMIN rule 403s any forced call. Documented non-goal (UI-SPEC §6). |
| T-2006-02 | Information Disclosure (XSS) | userAgent/ipAddress rendered in the Sessions DataTable | mitigate | Render via `{{ }}` interpolation (auto-escaped); explicitly no `v-html` for these attacker-influenceable strings (RESEARCH Security Domain). |
| T-2006-03 | Denial of Service (self) | revoke-others | mitigate | `sessionsApi.revokeOthers` (from 20-01) sends the live refreshToken so the current device is excluded; the control is only shown when >1 session exists. |
</threat_model>
