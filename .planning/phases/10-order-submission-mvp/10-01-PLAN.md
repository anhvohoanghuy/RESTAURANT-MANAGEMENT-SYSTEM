# Plan 10-01: Implement Order Submission From Cart With Table Snapshot And Kafka Event

<objective>
Add submitted order persistence and APIs so an authenticated user can submit their active cart into an order that preserves table and line snapshots, then publishes an order-created Kafka event for future consumers.
</objective>

<tasks>

1. Add submitted order model and DTOs.
   - Create order status enum if needed, order response DTOs, and stable submission error codes.
   - Include table snapshot fields on order response: `tableId`, `tableCode`, `tableName`, `areaId`, `areaName`.
   - Acceptance: order DTOs are separate from cart DTOs.

2. Add JPA persistence.
   - Add `orders`, `order_lines`, and order line topping snapshot persistence.
   - Store `userId`, status, submitted timestamp, table snapshot, line snapshots, and totals.
   - Acceptance: submitted order data remains readable after cart is cleared.

3. Implement order submission service.
   - Load authenticated user's active cart.
   - Reject empty cart with stable error.
   - Reject cart without table with stable error.
   - Copy cart table snapshot into order.
   - Copy all cart lines and topping snapshots into order.
   - Calculate total from copied line totals.
   - Clear active cart after successful save.
   - Acceptance: no menu/table re-validation or re-quote during submission.

4. Add owner-scoped read service methods.
   - List current user's submitted orders.
   - Get one current-user-owned order by id.
   - Acceptance: another user's order is not readable.

5. Add controller and security.
   - `POST /orders`
   - `GET /orders`
   - `GET /orders/{orderId}`
   - Use `@AuthenticationPrincipal CustomUserDetails`.
   - Protect `/orders/**` for authenticated `USER`/`ADMIN`.

6. Add Kafka order-created event publishing.
   - Add Spring Kafka producer dependency/config if absent.
   - Define an `OrderCreatedEvent` DTO with stable schema.
   - Add an `OrderEventPublisher` port and Kafka adapter.
   - Publish to topic property `order.events.order-created-topic`, default `orders.created`.
   - Publish only after order persistence succeeds.
   - Acceptance: no Kafka consumer is introduced in this phase.

7. Add focused tests.
   - Submit cart persists `tableId` and full table snapshot.
   - Submit cart persists dish/topping/quantity/price snapshots.
   - Submit empty cart fails with stable code.
   - Submit cart without table fails with stable code.
   - Successful submit clears active cart.
   - User cannot read another user's order.
   - Successful submit invokes event publisher with expected order id, user id, table snapshot, line snapshots, and total.
   - Failed submit does not publish an event.

8. Run full Maven tests.
   - Command: `mvnw.cmd test` with local `JAVA_HOME` if needed.

</tasks>

<verification>

- Focused Order Context tests pass.
- Existing cart, menu, table, and auth tests still pass.
- Full `mvnw.cmd test` passes.
</verification>

<success_criteria>

- Submitted orders exist separately from active carts.
- Order stores table id and table snapshot.
- Order stores immutable line snapshots.
- Cart submission rejects empty/no-table carts.
- Successful submission clears the active cart.
- Order reads are authenticated and owner-scoped.
- Successful submission publishes an `OrderCreated` Kafka event with stable payload.
- No payment/fulfillment/table occupancy behavior is introduced.
</success_criteria>
