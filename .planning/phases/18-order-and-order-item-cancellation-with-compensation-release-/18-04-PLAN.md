<objective>
Build Payment's FIRST-ever Kafka consumer and idempotency ledger to auto-refund a cancelled paid
order. On a whole-order `OrderCancelledEvent`, iterate every `PaymentEntity` for the order and, for
each, refund the unrefunded remainder by reusing the existing `PaymentService.recordRefund`
(inheriting its overpay guard and per-payment idempotency-key dedup) with a null system actor.
Partial-cancel events (`wholeOrder=false`) trigger no refund (D-6). Idempotent via a new
`payment_processed_events` ledger, wired with the exact cross-context DLT/poison-pill config used
elsewhere (including a dedicated DLT `KafkaTemplate` and `@EnableKafka`, since Payment has neither
today).

Purpose: Closes CANCEL-06 (automatic event-driven Payment refund, whole-order only). Depends only
on the Plan 01 event contract; runs in parallel with Plans 02/03 (payment_context files are
disjoint).
Output: `PaymentProcessedEventEntity`/repo, nullable `PaymentRefundEntity.actorUserId`,
`PaymentAutoRefundService`, listener, consumer config, and `PaymentAutoRefundServiceTest`.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-04-01 | Tampering / Repudiation (financial) | double-refund on redelivery | mitigate | Two independent layers: new `payment_processed_events` ledger by eventId + `recordRefund`'s existing `findByPayment_IdAndIdempotencyKey` dedup on the deterministic `auto-cancel-{orderId}-{paymentId}` key |
| T-18-04-02 | Tampering (financial) | over-refund exceeding payment amount | mitigate | Reuse `PaymentService.recordRefund`, whose `refundExceedsPayment` guard is authoritative; consumer only requests the computed unrefunded remainder |

| T-18-04-04 | Denial of Service | poison-pill event payload | mitigate | `ErrorHandlingDeserializer` + non-retryable `DeserializationException` → dedicated Payment DLT template |

| T-18-04-SC | Tampering | dependency installs | accept | No new dependencies |
