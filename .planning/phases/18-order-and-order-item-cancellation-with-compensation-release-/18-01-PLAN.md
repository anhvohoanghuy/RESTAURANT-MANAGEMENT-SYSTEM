<objective>
Establish the shared foundation every downstream cancel/compensation plan depends on: the terminal
`OrderStatus.CANCELLED` value, the two terminal-status guards that protect a cancelled order from
being resurrected by late saga events, the per-line `cancelledAt` soft-cancel marker, a
pessimistic row lock on `OrderRepository`, new cancel domain error codes, and the
`OrderCancelledEvent` outbox contract (plus its topic property) that Inventory, Payment, and
Kitchen consumers will subscribe to.

Purpose: This plan owns the cross-cutting order_context model/contract changes so that the Wave 2
consumer plans compile against a stable event class and the cancel service (Plan 02) has the enum,
marker, lock, and error codes it needs. It closes CANCEL-07 (terminal status + guards).
Output: Modified `OrderStatus`, `OrderDomainException`, `OrderLineEntity`, `OrderRepository`,
`KitchenStatusProjectionService`, `application.properties`; new `OrderCancelledEvent`; regression
tests.
</objective>
