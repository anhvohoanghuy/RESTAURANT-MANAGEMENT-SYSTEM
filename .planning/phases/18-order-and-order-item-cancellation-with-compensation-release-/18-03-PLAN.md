<objective>
Add the Inventory-side compensation consumer that releases held reservations for cancelled order
lines — the exact structural inverse of `InventoryReservationSettlementService`. On each
`OrderCancelledEvent`, for every cancelled `orderLineId`, re-resolve the line's recipe requirements
(reusing the existing resolver + lookup port, never reading the per-ingredient-aggregated
reservation lines), decrement `reservedQuantity` ONLY (never `quantityOnHand`) under a pessimistic
reservation lock with a non-negative clamp, write a new `RESERVATION_RELEASE` audit movement with a
null system actor, and record a per-line release ledger row. Idempotent via the majority
ledger-last-in-tx idiom plus a per-(orderId,orderLineId) guard.

Purpose: Closes CANCEL-05 (inventory reservation release, idempotent). Depends only on the Plan 01
event contract; runs in parallel with Plans 02/04 (disjoint files).
Output: `RESERVATION_RELEASE` movement type, `ReservationStatus.RELEASED`,
`InventoryLineReleaseEntity`/repo, `InventoryReservationReleaseService`, a widened settlement
completion guard, listener + consumer config, and `InventoryReservationReleaseServiceTest`.
</objective>
