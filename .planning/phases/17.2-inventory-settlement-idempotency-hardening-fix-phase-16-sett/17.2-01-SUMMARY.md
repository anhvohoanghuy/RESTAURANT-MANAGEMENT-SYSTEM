# Phase 17.2 Plan 01 Summary

## Objective

Close the Phase 16 inventory settlement idempotency risk where the processed-event ledger row was committed in a separate `REQUIRES_NEW` transaction before settlement work completed.

## Changes

- Removed `InventoryLedgerWriter` from `InventoryReservationSettlementService`.
- Moved settlement processed-event recording to the final durable step of the same `@Transactional` settlement method.
- Deleted the obsolete `InventoryLedgerWriter` production class and dedicated test.
- Added regression coverage for successful final ledger recording order and duplicate ledger failure propagation.
- Preserved existing duplicate `eventId` and per-order-line settlement guards.
